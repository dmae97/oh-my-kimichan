# OMK 16GB RAM 최적화 모드 — 2026-05-01

## 목표

16GB RAM 노트북/WSL 환경에서 `omk chat`, `omk run`, `omk team`, MCP 서버가 불필요하게 전역 MCP/스킬과 큰 출력 버퍼를 붙잡지 않도록 기본 실행량을 줄인다.

## 적용된 기본값

- `resource_profile = "auto"`: 호스트 RAM이 18GB 이하이면 `lite`로 동작한다.
- `mcp_scope = "project"`: lite 모드에서는 프로젝트 `.omk/.kimi` MCP만 주입하고 `~/.kimi/mcp.json` 전역 MCP는 기본 주입하지 않는다.
- `skills_scope = "project"`: lite 모드에서는 프로젝트 스킬만 주입하고 대량의 전역 스킬 로딩을 피한다.
- `max_workers = 1`: `omk run --workers auto`, `omk team --workers auto`의 기본 worker 수를 1로 낮춘다.
- `max_output_mb = 4`: shell/quality-gate 출력 버퍼를 4MiB로 제한한다.
- `wire_output_mb = 1`: wire task별 보관 출력을 1MiB로 제한한다.
- lite 모드에서는 이미지 로고 pre-render를 생략하고 ASCII 배너를 사용한다.

## 수동 오버라이드

```bash
# 강제 lite
OMK_RESOURCE_PROFILE=lite omk chat

# 전역 MCP/스킬까지 모두 로드
OMK_MCP_SCOPE=all OMK_SKILLS_SCOPE=all omk chat

# 출력 버퍼/worker 조정
OMK_MAX_OUTPUT_MB=8 OMK_WIRE_OUTPUT_MB=2 OMK_MAX_WORKERS=2 omk run feature-dev "..."
```

## 확인 방법

```bash
omk doctor
omk hud
```

두 명령 모두 현재 OMK runtime profile, MCP/skills scope, 출력 버퍼 크기를 표시한다.
